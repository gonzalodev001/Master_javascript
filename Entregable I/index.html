Ejercicios entregables de JavaScript nativo

Ejercicios

  1. [2pt] Implementa las siguientes funciones sobre arrays sin utilizar las funciones nativas de javascript:

     // Filtrar duplicación
 unique(['Salva', 'Sandra', 'Manuel', 'Daniel', 'salva', 'Sandra'], explicit);
 // explicit = false => ['Salva', 'Sandra', 'Manuel', 'Daniel']
 // explicit = true => ['Salva', 'Sandra', 'Manuel', 'Daniel', 'salva']

 // array unidimensional
 flatArray(['José', 'Manuel', ['María', 'Cristina'], ['Lucía', ['Luis', 'Carlos']]]);
 // ['José', 'Manuel', 'María', 'Cristina', 'Lucía', 'Luis', 'Carlos']

 // Revertir
 reverse(['a', 'b', 'c', 'd', 'e', 'f']);
 // ['f', 'e', 'd', 'c', 'b', 'a']

 // Ordenar
 const list = [
   { name: 'Dani', age: 20 },
   { name: 'Andres', age: 30 },
   { name: 'Marc', age: 40 },
   { name: 'Marc', age: 18 },
   { name: 'Isidoro', age: 25 },
 ];
 orderBy(list, 'name');
 /* [
 { name: 'Andres', age: 30 },
 { name: 'Dani', age: 20 },
 { name: 'Isidoro', age: 25 },
 { name: 'Marc', age: 40 },
 { name: 'Marc', age: 18 },
]; */

 // También puede aceptar varios parámetros
 orderBy(list, 'name', 'age');
 /* [
 { name: 'Andres', age: 30 },
 { name: 'Dani', age: 20 },
 { name: 'Isidoro', age: 25 },
 { name: 'Marc', age: 18 },
 { name: 'Marc', age: 40 },
]; */

// En caso de pasar un parámetro que no existe hay que devolver un error
    • Con funciones nativas me refiero a flat(2), reverse(3), sort(4)
    • unique el parámetro explicit es boolean, por defecto es false, en este caso la diferencia de mayúscula o minúscula para los textos no cuenta, en caso de ser true se debe tener en cuenta.
    • La función orderBy debe aceptar varios parámetros para ordenar, en caso de pasar un parámetro que no existe la función debe devolver un error.

Restricciones

Todos los ejercicios deben completarse utilizando únicamente código nativo de JavaScript, preferiblemente ECMAScript 2015.

Todo el código proporcionado debe estar escrito por el alumno (no pueden utilizarse librerías externas).

Se valorará tanto el grado de resolución del ejercicio como la calidad de la implementación (aprovechamiento de las funciones del lenguaje, legibilidad y simplicidad del código...).
